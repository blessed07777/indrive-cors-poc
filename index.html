⚠ inDrive — CORS Vulnerability PoC

Security Research · Bug Bounty · HackerOne · Authorized testing only

This page demonstrates CVE-class CORS misconfiguration in inDrive API. The pf-gw-cf.euce1.indriverapp.com profile endpoint reflects any Origin header with Access-Control-Allow-Credentials: true, allowing any website to read authenticated user data.

How the Attack Works

1
Victim is authenticated at couriers.indrive.com (web portal for inDrive couriers)
2
Victim clicks a link and opens this page on an attacker-controlled domain
3
JavaScript makes a cross-origin fetch() to the inDrive profile API with credentials: include
4
inDrive server reflects the attacker's origin in Access-Control-Allow-Origin
5
Browser allows JavaScript to read the full response — victim's PII is now in attacker's hands
