⚠️ inDrive — CORS Vulnerability PoC

Security Research Demo · Bug Bounty Report · For authorized testing only

⚠️ This page demonstrates a CORS misconfiguration vulnerability found in inDrive's API. It reads the profile of any logged-in inDrive user who visits this page — without any additional interaction. Submitted to HackerOne under inDrive's bug bounty program.

How the attack works

1
Victim is logged into inDrive app (JWT stored in browser/app session)
2
Victim clicks a link and opens this page (attacker-controlled domain)
3
JavaScript on this page makes a fetch() to inDrive's profile API
4
inDrive server responds with Access-Control-Allow-Origin: [this page's origin] + Access-Control-Allow-Credentials: true
5
Browser allows the response to be read — attacker receives victim's PII

Vulnerable endpoint

pf-gw-cf.euce1.indriverapp.com/api/profile-api/v1/profile
Origin: —
Access-Control-Allow-Origin: [reflects any origin]
Access-Control-Allow-Credentials: true
